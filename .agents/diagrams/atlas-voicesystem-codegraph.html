What the graph reveals

✓ Core isolation invariant holds The graph initially showed one core → adapters edge. Verified by hand: it's a resolver artifact — a local const text in server.ts name-collided with text() in adapters/pi/voice-line.ts. Core imports nothing from any adapter; its only "adapters" mention is a comment.
Legacy tree is a drifting mirror claudecode/.claude/PAI/USER/Voice/hooks/lib/ carries byte-for-byte symbol twins of the adapter libs (identity: 25 = 25, validators: 17 = 17). Two copies of the same graph cluster — every edit to one silently strands the other.
core/server.ts is the gravity well 95 symbols in one file — 21% of the entire graph — with zero exports. Everything reaches it over HTTP (POST /notify), never by import. Good for decoupling; bad for unit-testing the provider chain and circuit breakers locked inside.
Adapters never touch each other Zero static edges between adapters/pai and adapters/pi. Each cluster is internally cohesive (pi: index → config/voice-line/notify-client; pai: hooks → lib/) and converges only at the core's HTTP boundary — the architecture's promise, confirmed by the data.
